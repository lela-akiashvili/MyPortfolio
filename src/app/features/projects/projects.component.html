Pocket Library

Used: HTML, CSS, Javascript

GitHub Code: here

Site Description:
This site is built with HTML, CSS, and JavaScript, with a strong emphasis on functionality.
The main purpose is to allow visitors to search for movies of interest based on the names of books they input. Additionally, the site offers users the ability to find books by filtering and bookmarking them, even without signing in. Authorization and authentication are simulated using local storage as a database, enabling different users to maintain their personal libraries.
For this project, I utilized two separate APIs: the Open Library API and the OMDb API. However, since the OMDb API is not secure, obtaining movie results is only possible when the site is run on a local host.

Sport Today

Used: HTML, CSS, TypeScript, Angular, Firestore Database

GitHub Code: here

Site Description:
This site is dedicated to sports news, featuring mock data with fictional characters. It enables users to scroll through the latest sports news and filter it based on search input, sport, or creation date.
Signed-in users can favorite news articles to read later. Registered users can also write, publish, and delete their own news articles, with email verification ensuring the authenticity of registrations.
The site mimics and extends the user hierarchy found in the Teams application, offering four types of users: Regular User, Team's Coach, Manager, and Player. Each user type has different capabilities:

  • Coach: Can add or delete players and managers (this functionality needs to be correctly implemented).
  • Manager: Can plan games and write announcements for the team.
  • Player: Can read announcements and view games on their profile page.
    • This project extensively uses RxJS for stream control and database connections, as well as reactive forms for various functionalities.
